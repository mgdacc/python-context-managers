El Problema: Gestión de Recursos

Recursos como archivos o conexiones de red deben ser cerrados siempre, incluso si ocurre un error.

f = open("archivo.txt", "w")
f.write("Hola")
# ¿Qué pasa si aquí ocurre un error?
f.close() # Esta línea nunca se ejecutaría
2 / 14

La Solución Clásica: try...finally

El bloque `finally` garantiza que el código de limpieza se ejecute siempre, pero es verboso.

f = open("archivo.txt", "w")
try:
    f.write("Hola")
finally:
    f.close() # Se ejecuta sí o sí
3 / 14

La Solución "Pythónica": with

La declaración `with` simplifica esto enormemente. Garantiza que el recurso se cerrará automáticamente al salir del bloque.

with open("archivo.txt", "w") as f:
    f.write("Hola")
# El archivo se cierra automáticamente aquí

¿Qué es un Context Manager?

Un objeto diseñado para usarse con la declaración `with`. Define qué hacer al "entrar" en el bloque y qué hacer al "salir".

5 / 14

Crear un Context Manager (Clase)

Para crear uno, necesitas una clase con dos métodos dunder: __enter__ y __exit__.

6 / 14

Método: __enter__

Se ejecuta al inicio del bloque `with`. Su valor de retorno es lo que se asigna a la variable después de `as`.

class MiContexto:
    def __enter__(self):
        print("Entrando al contexto...")
        return "Valor" # Esto va a la variable 'v'
7 / 14

Método: __exit__

Se ejecuta al final del bloque `with`, sin importar cómo se salga (normalmente o por un error). Ideal para la limpieza.

class MiContexto:
    # ... __enter__ ...
    def __exit__(self, exc_type, exc_val, exc_tb):
        print("Saliendo del contexto...")
        # Los argumentos sirven para manejar excepciones si ocurrieron
8 / 14

Ejemplo: Un Temporizador

Un context manager perfecto para medir el tiempo de un bloque de código.

import time

class Timer:
    def __enter__(self):
        self.start = time.time()
        return self # No necesitamos devolver nada especial
    
    def __exit__(self, *args):
        end = time.time()
        print(f"El bloque tardó {end - self.start:.4f}s")
9 / 14

Uso del Temporizador

Ahora podemos "envolver" cualquier código con nuestro `Timer`.

with Timer():
    # Código que queremos medir
    time.sleep(2)

Forma Fácil: @contextmanager

El módulo `contextlib` ofrece un decorador para crear context managers a partir de una función generadora, de forma más simple.

11 / 14

La Lógica de @contextmanager

Todo lo que está antes de `yield` es el `__enter__`. Todo lo que está después de `yield` es el `__exit__`.

from contextlib import contextmanager
import time

@contextmanager
def timer():
    # --- Código de __enter__ ---
    start = time.time()
    yield # Aquí se ejecuta el código dentro del 'with'
    # --- Código de __exit__ ---
    end = time.time()
    print(f"El bloque tardó {end - start:.4f}s")
12 / 14

Uso del Decorador

Se usa exactamente igual que la versión con clase.

with timer():
    time.sleep(1.5)
13 / 14

Conclusión

Los Context Managers y la declaración `with` son la forma idiomática y segura de gestionar recursos en Python, garantizando que las tareas de limpieza siempre se ejecuten.
